# Airbnb MCP Project Rules

## Airbnb 검색은 반드시 서브에이전트에게 위임

openbnb MCP 도구(`mcp__openbnb__airbnb_search`, `mcp__openbnb__airbnb_listing` 등)를 사용할 때는 **메인 스레드에서 직접 호출하지 말 것**. 검색 결과가 토큰을 대량으로 소비하여 메인 컨텍스트를 오염시킨다.

### 방법
- `Agent` 도구로 서브에이전트를 생성하여 검색을 위임한다
- 모델은 `haiku`를 지정한다 (비용 효율)
- 서브에이전트 프롬프트에 검색 조건과 **응답 형식**(요약 테이블, 항목 수 제한 등)을 명시한다
- 서브에이전트가 결과를 정리하여 간결하게 보고하면, 메인 스레드에서 사용자에게 전달한다

### 예시 패턴
```
Agent({
  description: "서울 에어비앤비 검색",
  model: "haiku",
  prompt: "openbnb MCP로 서울 에어비앤비를 검색해줘. 조건: 체크인 2026-06-01, 체크아웃 2026-06-03, 성인 2명, 1박 $100 이하. 결과를 숙소명 / 위치 / 평점 / 2박 총 가격 / 배지 형태의 마크다운 테이블로 상위 5개만 정리해서 보고해."
})
```

### 예외
- 사용자가 명시적으로 메인 스레드에서 호출을 요청한 경우
- 단건 리스팅 상세 조회(`airbnb_listing`)로 결과가 작을 것이 확실한 경우
